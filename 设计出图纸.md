# 设计出图纸

## 项目名称

室内设计：从 CAD 到 3D 效果图设计

## 当前任务目标

请先不要开发网页、App、家具布局、风格生成、灯光渲染和完整装修效果图。

当前阶段只做一件事：

> 稳定跑通 CAD/DXF 标准化 → 空间语义解析 → 诊断报告 → Blender 毛坯房生成。

核心原则：

> Blender 不直接理解 CAD。Blender 只读取 `project.json` 中已经明确标注的空间语义。

## 第一优先级：保证 CAD 进入 Blender 前已经语义化

请重点检查并优化以下内容：

### 1. 图层标准化

输入 CAD/DXF 图层可能很乱，例如：

```text
WALL
墙体
A-WALL
建筑墙线
DOOR
门
WINDOW
窗
ROOM_TEXT
房间名称
```

请通过 `config/layers.json` 或代码内映射，把它们统一成：

```text
WALL
DOOR
WINDOW
ROOM_TEXT
```

非核心图层不要进入后续流程，尤其要删除或忽略：

```text
家具
吊顶
灯具
尺寸
标注
水电
开关
插座
轴线
图框
填充
说明文字
```

---

### 2. 强制压平 CAD

很多 CAD 不是纯 2D，可能有 Z 值。

请确保标准化后的 DXF 中用于建模的实体全部压平到：

```text
z = 0
```

Blender 中的墙高、门高、窗高必须由代码生成，不要相信 CAD 原始 Z 值。

默认高度：

```text
墙高 = 2800mm
门高 = 2100mm
窗台高 = 900mm
窗高 = 1200mm
```

---

### 3. 单位统一

`project.json` 内部统一使用：

```text
mm
```

进入 Blender 时统一转换：

```text
1 Blender unit = 1 meter
mm → m = 除以 1000
```

请检查是否存在单位异常：

```text
户型最大宽度 5~30：可能原图单位是米
户型最大宽度 5000~30000：可能原图单位是毫米
户型最大宽度 > 100000：可能坐标过大或单位异常
```

如果无法确定单位，不要静默继续，应该在诊断报告里输出 warning。

---

### 4. 坐标归一化

CAD 坐标可能离原点很远，例如：

```text
x = 3500000
y = 2400000
```

请在解析阶段归一化：

```text
所有有效点减去 min_x, min_y
```

同时在 `project.json` 中保存原始偏移：

```json
"origin_offset": [min_x, min_y]
```

避免 Blender 中模型看不到、相机异常、精度抖动。

---

### 5. 处理 INSERT / BLOCK

门窗在 CAD 中经常不是 LINE，而是：

```text
INSERT / BLOCK
```

请不要丢弃 `INSERT`。

至少需要保留：

```text
block name
insert point
rotation
xscale
yscale
source layer
```

第一版可以先把 INSERT 简化为门窗中心点和宽度，但不能直接忽略。

## 第二优先级：生成诊断报告，避免错误传到 Blender

请确保每次运行后都有这些文件：

```text
output/diagnostics/dxf_report.json
output/diagnostics/project_report.json
output/diagnostics/plan_preview.png
```

### dxf_report.json 要包含

```text
输入文件路径
输出标准化文件路径
图层统计
实体类型统计
是否存在 WALL
是否存在 DOOR
是否存在 WINDOW
是否存在 ROOM_TEXT
是否发现异常 Z 值
是否发现坐标过大
是否还有疑似家具/吊顶/标注图层
errors
warnings
```

### project_report.json 要包含

```text
walls 数量
doors 数量
windows 数量
rooms 数量
户型 bounding box
单位判断
门是否能匹配最近墙
窗是否能匹配最近墙
房间名称是否识别
是否需要人工确认
errors
warnings
```

### plan_preview.png 要显示

```text
墙 = 红色
门 = 蓝色
窗 = 绿色
房间文字 = 紫色
```

如果 `plan_preview.png` 中平面图不对，就不要继续调 Blender。

## 第三优先级：确保 Blender 只读取 project.json

请检查 `scripts/02_build_shell_blender.py`：

1. 不要直接导入 DXF。
2. 只读取 `output/project.json`。
3. 根据 `walls` 生成墙体。
4. 根据 `rooms` 生成地面。
5. 根据 `doors` / `windows` 生成门窗标记或洞口占位。
6. 给 Blender 对象写入语义属性：

```text
semantic_type = wall / door / window / floor
source_id = 对应 project.json 中的 id
```

这样后续家具布局、风格材质、灯光渲染才能继续接。

## 运行命令

先跑安全流程，不进 Blender：

```bash
pip install -r requirements.txt
python run_phase1_2.py input/house.dxf --skip-blender
```

检查输出：

```text
output/standardized/house_standardized.dxf
output/project_raw.json
output/project.json
output/diagnostics/dxf_report.json
output/diagnostics/project_report.json
output/diagnostics/plan_preview.png
```

确认 `plan_preview.png` 正确后，再跑 Blender：

```bash
python run_phase1_2.py input/house.dxf
```

如果本机没有 Blender，请不要阻塞任务。先保证 Phase 0、Phase 1、诊断报告和 2D 预览图能正确生成。

## 设计原则

请始终遵守：

```text
CAD 只负责提供原始几何。
标准化阶段负责清理和保留核心图层。
解析阶段负责生成语义数据。
诊断阶段负责阻止错误继续传递。
Blender 阶段只负责根据语义数据建模。
```

最终目标不是让 Blender 理解 CAD，而是让 Blender 执行已经定义好的空间语义。
